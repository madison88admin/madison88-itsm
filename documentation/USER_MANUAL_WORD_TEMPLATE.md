## 2. Glossary

| Term             | Definition                                                      |
|------------------|-----------------------------------------------------------------|
| Agent            | A person who handles IT tickets and requests.                   |
| End User         | Any employee or person submitting a ticket or request.          |
| SLA (Service Level Agreement) | The expected response and resolution time for a ticket. |
| Ticket           | A record of an IT issue, request, or question.                  |
| Knowledge Base   | A collection of articles and solutions for common problems.     |
| Escalation       | Moving a ticket to a higher support level or manager.           |
| Priority         | The urgency of a ticket (P1 = Critical, P2 = High, etc.).       |
| Dashboard        | A summary view of system status, tickets, and performance.      |
| Attachment       | A file (image, document, etc.) added to a ticket.               |
| Queue            | A list of tickets assigned to an agent or team.                 |
| Resolution       | The solution or fix for a ticket.                               |
| Internal Note    | A comment visible only to IT staff, not end users.              |
| Public Comment   | A message visible to both IT staff and end users.               |

## 3. Troubleshooting Flowchart

**Troubleshooting Steps:**

1. Is your issue urgent (system down, security risk)?
	- Yes → Create a ticket and mark as "Critical" (P1)
	- No → Go to step 2
2. Can you find a solution in the Knowledge Base?
	- Yes → Follow the article steps
	- No → Go to step 3
3. Do you need to attach files/screenshots?
	- Yes → Attach them when creating your ticket
	- No → Go to step 4
4. Submit your ticket and wait for updates.
	- Check your email or the platform for notifications
5. If no response within SLA, escalate to your manager.

For more help, contact IT support at it-support@madison88.com.

## 6. End User Guide

### Creating a Ticket
[Insert Screenshot: New Ticket Form]

1. Open the left navigation menu
2. Click New Ticket
3. (Optional) Select a Ticket Template to prefill common fields
4. Complete the required information
5. Click Submit

### Tracking Your Tickets
[Insert Screenshot: Ticket Status Page]

1. Click Tickets from the menu
2. View ticket status, assigned agent, and SLA countdown
3. Click a ticket to open its detailed view

### Communicating on Tickets
[Insert Screenshot: Ticket Comments Section]

1. Open the ticket
2. Scroll to the Comments section
3. Enter your message
4. Click Post Comment

### Uploading Attachments
[Insert Screenshot: Add Attachment Button]

1. Open the ticket detail page
2. Locate the Attachments section
3. Click Add Attachment
4. Select a file
5. Click Upload

### Using the Knowledge Base
[Insert Screenshot: Knowledge Base Article]

1. Click Knowledge Base from the menu
2. Use the search bar to find relevant articles
3. Apply category filters if needed
4. Open an article to view the solution
